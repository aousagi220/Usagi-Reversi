# USAGI REVERSI - プロジェクト概要

> ChatGPT向けのプロジェクト説明ドキュメントです。

---

## プロジェクト概要

**名称:** USAGI REVERSI（うさぎリバーシ）
**種別:** Webブラウザで動作するリバーシゲーム
**技術スタック:** HTML / CSS / Vanilla JavaScript

---

## 今ある機能

### ゲームロジック

| 機能 | 詳細 |
|------|------|
| ボード管理 | 8×8 のマスを2次元配列で管理。ゲーム開始時に中央4マスへ初期石を配置 |
| 石の配置検証 | クリックしたマスに置けるか 8方向（上下左右＋斜め4方向）をチェック |
| 石の反転 | 挟んだ相手の石を即時に反転。1ターンで複数方向の反転にも対応 |
| ターン管理 | 黒→白→黒... と自動で手番を切り替え |
| パス処理 | 置ける場所がないプレイヤーは自動でパス（相手のターンへ） |
| ゲーム終了判定 | 両プレイヤーが連続でパスになったらゲーム終了と判定 |
| 勝敗判定 | 終了時に石の数を比較し「黒の勝ち / 白の勝ち / 引き分け」を判定 |
| スコア集計 | 盤面の黒石・白石の総数をリアルタイムで集計 |

### UI / 表示

| 機能 | 詳細 |
|------|------|
| ボード描画 | CSS Grid で 8×8 のセルを動的生成 |
| 有効手ハイライト | 現在の手番で置けるマスを緑色で表示 |
| スコアパネル | 黒・白それぞれの石数と現在の手番をリアルタイム表示 |
| 勝利モーダル | ゲーム終了時に勝者・最終スコア・再戦ボタンをモーダルで表示。出現時にアニメーションあり |
| リセット / 再戦ボタン | ゲームを初期状態に戻す（サイドパネルのリセットボタンと勝利モーダルの再戦ボタンの2箇所） |
| レスポンシブ対応 | デスクトップ・タブレット・モバイルの3段階で画面サイズに対応 |

## 次に実装する機能

### パス通知UI

現状、パスが発生しても手番が切り替わるだけで画面上に何も表示されない。
プレイヤーがなぜ手番が変わったか分からないため、パスされたことを画面で知らせるUIを追加する。

**実装イメージ**

- パス発生時にトースト通知（数秒で自動的に消えるポップアップ）を表示
- 「〇〇はパスされました」などのメッセージを出す
- `script.js` 内の `console.log("パスされました！")` の箇所でUI関数を呼ぶ形で実装

**役割分担（現在の設計に合わせた方針）**

| ファイル | やること |
|----------|----------|
| `index.html` | トースト用の要素を追加 |
| `index.css` | トーストの見た目・表示アニメーションを追加 |
| `UI.js` | トーストを表示する関数を追加 |
| `script.js` | パス発生時に `UI.js` の関数を呼ぶ |

## 将来的に実装してみたい機能

### CPU / AI 対戦

人間 vs コンピュータで対戦できるモードを追加したい。

**検討中のアイデア**

| レベル | アルゴリズム案 |
|--------|----------------|
| 弱い CPU | ランダムに置ける場所を選ぶ |
| 普通の CPU | 最も多く石を取れるマスを選ぶ（貪欲法） |
| 強い CPU | 角を優先・危険マスを避けるヒューリスティック |
| AI | ミニマックス法 / アルファベータ枝刈りなど |

**課題・検討点**

- CPUの思考タイミング（非同期処理 or `setTimeout`）
- 難易度選択UIの設計
- 将来的に機械学習（強化学習）を使ったAIも検討したい
